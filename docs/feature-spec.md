# 功能规格说明

> 本文档定义每个模块的详细行为、交互边界和数据结构。
> 标注 `[MVP]` 为第一版必须实现；`[P1]` 为第二版迭代。

## 1. 番茄钟模块 🍅

### 1.1 状态机

```
                 ┌─ 开始 ──→ [running]
                 │              │  │
              [idle]    暂停 ←──┘  │ 倒计时归零
                 │    (pause)      │
                 ↑        │       ↓
                 │   继续 │   [finished]
                 │        ↓       │
                 └── 放弃 ←───────┘
              (give up)
```

| 状态 | 说明 |
|------|------|
| `idle` | 初始状态，未开始计时 |
| `running` | 计时中，倒计时递减 |
| `paused` | 已暂停，计时停止但不重置 |
| `finished` | 计时自然结束（到 00:00） |

### 1.2 计时规则 `[MVP]`

| 参数 | 默认值 | 可自定义范围 |
|------|--------|-------------|
| 专注时长 | 25 分钟 | 1 ~ 120 分钟 |
| 短休息 | 5 分钟 | 1 ~ 30 分钟 |
| 长休息 `[P1·部分完成]` | 15 分钟 | 5 ~ 60 分钟 |
| 长休息触发轮次 `[P1]` | 每 3 轮 | 2 ~ 6 轮 |

### 1.3 操作行为 `[MVP]`

| 操作 | 前置状态 | 效果 |
|------|---------|------|
| **开始** | idle / finished | 从设定时长开始倒计时，关联当前选中的任务 |
| **暂停** | running | 计时冻结，保留剩余时间 |
| **继续** | paused | 从剩余时间继续倒计时 |
| **放弃** | running / paused | 计时重置为设定时长，本次不计入统计 |

**规则**：
- 放弃的番茄**不计入**完成数，但已在 `PomodoroRecord` 中记录（`completed: false`）
- 暂停期间浏览器标签页切换到后台，计时器继续走（Web Worker 计时）
- 专注计时结束后自动标记关联任务为"已投入 X 个番茄"

### 1.4 通知 `[MVP]`

- 计时结束前 **5 秒**发出准备提醒（可选 `[P1]`）
- 计时归零时：
  - 浏览器 Notification API 弹出通知："🎉 专注完成！休息一下吧"
  - 播放提示音（默认铃声）
  - 页面标题闪烁提示（当标签页不在前台时）
- 用户在计时结束页面停留，显示"休息"或"继续下一轮"按钮

### 1.5 与待办关联 `[MVP]`

- 用户在待办列表中**选中**一个任务
- 点击"开始专注" → 番茄钟关联该任务
- 专注结束时，该任务的"投入番茄数" +1
- 未选中任务时也可以开始专注（通用番茄）

## 2. 待办事项模块 ✅

### 2.1 数据结构 `[MVP]`

```ts
interface Todo {
  id: string;               // 唯一标识 (nanoid)
  title: string;            // 任务标题 (1-200字符)
  completed: boolean;       // 是否完成
  pomodoroCount: number;   // 已投入番茄数
  createdAt: number;        // 创建时间戳
  completedAt: number | null; // 完成时间戳
  order: number;            // 排序序号
  dateKey: string;          // 所属日期 "2026-06-26"
}
```

### 2.2 交互行为 `[MVP]`

#### 添加任务
- 顶部/底部有输入框，placeholder："添加今天的任务…"
- 输入后按 Enter 或点击 ➕ 按钮添加
- 新任务出现在列表顶部（或底部，可配置 `[P1]`）
- 空标题（纯空格）不允许添加，输入框抖动提示
- 单日任务数上限 50 条（防止滥用）

#### 完成任务
- 点击任务左侧 ○ 圆圈 → 变为 ✓ 对勾
- 已完成任务文字添加删除线 + 灰色
- 已完成任务自动沉到底部
- 可以取消完成（点击 ✓ 恢复为 ○）

#### 编辑任务 `[MVP]`
- 双击任务文字 → 进入编辑模式（inline input）
- Enter 确认，Esc 取消
- 编辑后保留原有番茄数

#### 删除任务 `[MVP]`
- 右滑露出红色删除按钮（移动端）
- 或点击任务右侧 🗑 图标（桌面端）
- 删除前无确认弹窗（追求极简），但提供 3 秒撤销提示
- 删除任务不影响已记录的番茄统计

### 2.3 今日视图 `[MVP]`

- 默认显示"今日"标签
- 每日凌晨 0 点自动刷新为新的一天
- 未完成的任务自动滚动到新的一天（不丢失，可在"历史"查看 `[P1]`）

### 2.4 优先级 `[P1]`

- 每个任务可选优先级：🔴高 / 🟡中 / 🟢低（默认低）
- 列表按优先级排序 + 用户自定义拖拽排序

## 3. 统计模块 📊

### 3.1 基础统计 `[MVP]`

页面顶部或侧边展示：

| 指标 | 说明 |
|------|------|
| 🍅 今日番茄 | 今日完成的专注番茄数 |
| ✅ 今日完成 | 今日完成的任务数 / 总任务数 |

### 3.2 图表统计 `[P1]`

| 图表 | 维度 | 展示形式 |
|------|------|----------|
| 每日番茄数 | 最近 7 天 / 30 天 | 柱状图 |
| 累计专注时长 | 本周 / 本月 / 总计 | 数字 + 进度条 |
| 任务完成率 | 每日完成率趋势 | 折线图 |
| 专注时段分布 | 上午/下午/晚上 | 饼图或环形图 |
| 连续打卡天数 | 连续专注天数 | 数字 + 🔥 火焰图标 |

### 3.3 数据存储策略 `[MVP]`

- 所有数据存储在浏览器 `localStorage`
- 单键 `pomodoro_app_data`，JSON 格式
- 数据版本号，便于未来迁移
- 定期（每次保存）做数据完整性校验

## 5. 用户界面布局

### 5.1 桌面端布局 `[MVP]`

```
┌──────────────────────────────────────────────┐
│  🍅 极简番茄          📊 4 🍅  ✅ 3/5      │  ← 顶部状态栏
├──────────────────────┬───────────────────────┤
│                      │                       │
│    🍅 24:59          │   📋 今日待办         │
│                      │                       │
│    [暂停]  [放弃]     │   ○ 复习高数         │
│                      │   ✓ 做完英语真题      │
│    当前：复习高数      │   ○ 背50个单词       │
│                      │   ○ 整理笔记          │
│                      │                       │
│                      │   [添加任务...]  ➕   │
│                      │                       │
├──────────────────────┴───────────────────────┤
│  🏆 成就  │  📊 统计  │  ⚙️ 设置            │  ← 底部导航 `[P1]`
└──────────────────────────────────────────────┘
```

### 5.2 移动端布局 `[MVP]`

单列布局，番茄钟在上、待办列表在下，可滚动。
底部固定导航栏。

### 5.3 设计原则

- 大面积留白，减少视觉噪音
- 主色调：番茄红（#E74C3C）+ 白色背景
- 字体：系统默认无衬线字体
- 动画：仅用于计时器数字变化和完成勾选，克制使用
